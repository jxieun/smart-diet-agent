# 📝 ADR-009: 메뉴판 텍스트 추출을 위한 OCR 엔진 선정 및 구현

| **상태** | 승인됨 (Accepted) |
| **작성자** | J |

## 📋 상태
**Accepted**

## 🎬 1. 배경 (Context)
* 사용자로부터 업로드된 메뉴판 이미지에서 메뉴 이름과 가격 정보를 추출하여 디지털 데이터로 변환해야 합니다.
* 한국어와 영어 메뉴가 혼재된 이미지를 정확하게 인식할 수 있는 OCR(Optical Character Recognition) 솔루션이 필요합니다.

## 🏗️ 2. 기술적 결정 (Decision)
1. **EasyOCR 라이브러리 채택**:
    * **선정 이유**: 80개 이상의 언어를 지원하며, 특히 한글 인식률이 매우 높고 설정이 간편합니다.
    * **구현 방식**: PyTorch 기반의 딥러닝 모델을 사용하여 이미지 내 텍스트를 리스트 형식으로 추출합니다.
2. **싱글톤(Singleton) 서비스 구조**:
    * 서버 실행 시 OCR 모델을 한 번만 로드하여 메모리 효율성을 높이고 응답 속도를 최적화합니다.
3. **모노레포 간 파일 경로 최적화**:
    * `os.path.abspath`와 `os.path.join`을 활용하여 NestJS의 업로드 폴더와 FastAPI 분석 엔진 간의 상대 경로 문제를 해결했습니다.

## ⚖️ 3. 결과 (Consequences)
* **장점**: 
    * 별도의 클라우드 유료 API(Google Vision 등) 없이 로컬 환경에서 무료로 고성능 OCR을 구현했습니다.
    * 메뉴판 이미지에서 '비빔밥', '떡볶이' 등 한글 텍스트를 성공적으로 추출함을 확인했습니다.
* **단점**: 
    * PyTorch 모델 로드 시 초기 가동 시간이 다소 소요되며, CPU/GPU 자원을 상당 부분 점유합니다.
